# 🏠 AI기반 임대차계약서 검토 서비스

## 2. 프로젝트 개요

### 프로젝트명
**AI기반 부동산 임대차 계약서 검토 서비스 

### 프로젝트 소개
부동산 임대차 계약은 보증금, 특약, 계약 기간 등 꼼꼼히 따져야 할 내용이 많지만, 
법률 지식이 부족한 사회 초년생이나 일반 임차인이 스스로 모든 정보나 내용을 파악하기란 쉽지 않습니다.

본 서비스는 사용자가 임대차 계약과 관련된 상황이나 궁금증을 입력하면,
주택임대차보호법·판례·법령해석례를 검색하여 관련 내용을 안내하고
놓치기 쉬운 확인 사항을 함께 알려주는 **RAG 기반 LLM 챗봇**입니다.

사용자는 계약서를 업로드하거나 특약 내용을 직접 입력하여 해당 조항의 위험도를 확인하고,
상황별 특약 체크리스트를 통해 놓친 조항이 없는지 점검할 수 있습니다.

검색된 법령·판례·해석례 문서를 근거로 답변을 생성하므로,
**근거 없는 정보 제공(할루시네이션)을 방지**하고 모든 답변에 출처를 명시합니다.

> ⚠️ **본 서비스는 법률 자문이나 법적 조언을 제공하는 서비스가 아닙니다.**
> 주택임대차보호법·판례·법령해석례에 기반한 정보 안내 서비스이며,
> 구체적인 법적 판단이 필요한 경우 반드시 전문가와 상담하시기 바랍니다.

피해의 주요 원인은 **계약 전 위험 조항 미인지**와 **특약 내용에 대한 법적 효력 무지**입니다.
임대차 계약서에는 임차인에게 불리한 특약이 포함되는 경우가 많지만,
법률 지식이 부족한 일반인이 이를 계약 전에 스스로 파악하기는 매우 어렵습니다.


#### 현재 국내 부동산 관련 AI 서비스

(1) 공공 무료 AI 서비스
- **서울시 전세사기 위험분석 보고서**
: 2025/10/24부터 운영 중 (서울주거포털/내집스캔 연동)
* 특징: 주소 입력 시 집주인 신용 및 주택 위험도 24종 데이터 기반 보고서 즉시 발급
* 한계: 서울 소재 주택 한정, 쿠폰 방식의 수량 제한(인당 최대 2회 등)

- **경기도 AI 기반 거래 안전망 솔루션** 
: 2026년 상반기 시스템 완성 및 하반기 시범 운영 예정 (예산 14억 원)
* 특징: 실시간 등기부 변동 모니터링 및 사전 예방형 위험 경고 시스템
* 한계: 경기도 지역 한정 및 공인중개사 중심 연계, **현재 미출시 상태**

(2) 기존 민간 AI 서비스

* **C-node / 앨리비 / Law.ai**: 
    - B2B 중심: 주로 기업 법무팀의 업무 효율화(계약 관리, 시간 단축)를 목적으로 하며, 
    일반인이 1회성 계약 검토를 위해 접근하기 어려움.
    - 도메인 희석: 모든 계약서를 다루는 '범용성'에 치중하여, 임차인에게 치명적인 
    **임대차 독소 조항(특약)**에 대한 정밀한 RAG 대응이 부족함.
    
* **기타 서비스 (CheckDoc / 계약서마당)**:
    - 프리랜서 등 특정 대상에 국한되거나, 검토보다는 자동 '작성' 기능에 집중되어 있어 
    실질적인 법적 위험 판별이 어려움.

**→ 임대차 계약 내용을 법령·판례 근거로 질의응답할 수 있는 일반인 대상 서비스는 현재 존재하지 않음!**

본 프로젝트는 **임대차 계약서 내용과 특약**을 법령·판례 근거로 질의응답할 수 있는 
**국내 유일의 B2C 일반인 대상 B2C 대화형 AI 서비스**를 지향하며, 주소기반의 위험도 조회와 B2B 서비스의 
양분화 사이에서 그 공백을 메우는 독보적인 포지션을 가집니다.

#### 🚀 우리 프로젝트의 경쟁 우위 (Competitive Edge)

1. **"주소"가 아닌 "내용"에 집중**
   - 공공 서비스가 '이 집이 안전한가?'를 묻는다면, 우리는 **'내가 사인할 이 계약서의 문구가 나에게 안전한가?'**에 답합니다.

2. **임대차 특화 RAG (Retrieval-Augmented Generation)**
   - 일반 법률 데이터가 아닌, **주택임대차보호법(주임법), 실제 판례, 행정 해석례** 데이터를 활용하여 생성형 AI의 할루시네이션을 방지하고 정확한 법적 근거를 제시합니다.

3. **자연어 질의응답 (Interactive Q&A)**
   - 어려운 법 용어를 몰라도 "이 특약이 나중에 보증금을 돌려받을 때 방해가 될까요?"와 같이 일상적인 언어로 소통하며 위험 요소를 파악할 수 있습니다.



- 다양한 입력 방식 지원: 특약 직접 입력 / 상황 설명 / 계약 조건 질의
- 할루시네이션 방지: 내부 문서(법령·판례·해석례)에 근거한 답변만 생성
- 민감정보(주민번호, 주소 등) 자동 마스킹 처리
- 임차인이 계약서 독소조항·불리한 조항을 쉽게 파악할 수 있도록 지원
- 사용자가 업로드한 계약서 PDF를 분석하여 **맞춤형 리스크 진단** 제공
- Streamlit UI를 통한 **누구나 쉽게 접근 가능한** 서비스 구현



#### 비즈니스 모델 및 시장 가치

##### 타깃 사용자
- 처음 임대차 계약을 앞둔 **사회초년생·청년층**
- 재계약 또는 특약 검토가 필요한 **임차인**
- 계약서 검토 비용을 절감하고 싶은 **일반 세입자**

##### 시장 적합성
- 연간 전월세 거래량 약 **200만 건 이상** (국토교통부 실거래가 기준)
- 전세사기 피해 예방에 대한 **사회적 수요 급증**
- 기존 서비스 대비 차별점: **계약서 파일 기반 조항별 분석 + 법령 근거 + 체크리스트**

##### 수익 모델 (프로덕트 확장 방향)
- 무료 기본 서비스 + 프리미엄 구독 (심층 분석, 비교 검토)
- 공인중개사·부동산 플랫폼과의 B2B 제휴
- 정부·공공기관 임차인 보호 서비스 연계

## 4. 시스템 아키텍처
```
사용자 (Streamlit)
    │
    ├─ 계약서 업로드 (PDF/JPG)
    │       ↓
    │   OCR 텍스트 추출
    │       ↓
    │   민감정보 마스킹
    │
    └─ 질문 입력
            ↓
    질문 임베딩 (Embedding Model)
            ↓
    Qdrant VectorDB 검색 (Top-K)
    [법령 / 판례 / 법령해석례]
            ↓
    RAG 프롬프트 구성
            ↓
    LLM 답변 생성
            ↓
    위험 조항 분류 + 출처 포함 답변 반환
```
